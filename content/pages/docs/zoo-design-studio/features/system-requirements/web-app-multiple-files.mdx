The browser version of Zoo Design Studio supports multiple files in a single workspace. This brings
the web workflow closer to the desktop app for projects that need more than one KCL file.

## Multiple Files

Add and switch between multiple project files from the web workspace. This is useful for imported
KCL examples, generated Zookeeper projects, and assemblies or models split across more than one
source file.

## Project Files

In the browser version, project files are still stored in browser-managed storage rather than a
normal folder on disk. Use export or version-control workflows when you need durable local copies.

## Multi-Project Workflows

Multiple files in one workspace are supported, but full multi-project desktop workflows are still
different from browser workflows.

## Browser Choice

Use a modern browser for the web app. If an enterprise workflow depends on a specific browser, test
the file workflow there before standardizing on it.

## Are you interested in code?

Multiple-file projects use normal KCL imports between files.

```kcl
import "parts/bracket.kcl" as bracket

mainBracket = bracket
```

Want to find out more about KCL modules and imports? [Check it out](/docs/kcl-lang/modules) in our
KCL docs.
