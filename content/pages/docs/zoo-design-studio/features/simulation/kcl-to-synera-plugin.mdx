The KCL to Synera Plugin connects Zoo's parametric CAD workflow to Synera analysis workflows. Use it
to run a KCL project, export the resulting geometry as STEP, and pass that geometry into Synera for
simulation tasks such as FEA setup.

## KCL Project

Choose the KCL project that should feed the Synera workflow. The plugin runs the KCL project so the
simulation starts from regenerated parametric geometry, not a stale static export.

## STEP Export

The plugin exports the evaluated model as STEP before handing it to Synera. STEP is the preferred
format for downstream engineering and simulation tools because it preserves precise CAD geometry.

## Face And Body Metadata

Use names, colors, and tags in the Zoo model to make downstream selections easier. Preserved
metadata helps Synera workflows target bodies and faces without relying on fragile face indices.

## Multi-File Projects

KCL projects can contain multiple files with relative imports. Keep project structure stable so the
plugin can resolve the same files Zoo Design Studio uses.

## Enterprise Environments

Configure the plugin for the correct API host and licensing environment when working inside an
enterprise setup.

## Are you interested in code?

Zoo Design Studio uses KCL as the source of truth for generated geometry. A Synera workflow starts
from normal KCL, then the plugin evaluates and exports that model.

```kcl
@settings(defaultLengthUnit = mm)

mountPlate = extrude(mountProfile, length = 5mm)
analysisRib = extrude(ribProfile, length = 18mm)
```

Want to find out more about exportable KCL geometry? Start with the
[KCL docs](/docs/kcl) and the [Export](/docs/zoo-design-studio/features/data-management/export)
section.
